Claude가 대화 내용을 기억하고 학습할 수 있게 해주는 <b>하이브리드 메모리 시스템</b>입니다.

<b>Knowledge Graph Memory Server</b>의 안정성과 <b>Mem0 MCP</b>의 고성능 검색을 결합하여 최적의
메모리 환경을 제공합니다. GPT와 마찬가지로 대화 내용을 기억하고 새 대화방에서 메모리를 토대로 대화를
이어 나갈 수 있습니다.

---

## 📊 두 시스템 비교 분석

각 시스템의 특성을 이해하고 적절히 활용하는 것이 중요합니다.

| 구분            | Knowledge Graph Memory Server                        | Mem0 MCP                                                     |
| --------------- | ---------------------------------------------------- | ------------------------------------------------------------ |
| **저장 위치**   | 로컬 파일 시스템                                     | 클라우드 (Mem0 서버)                                         |
| **API 키**      | 불필요                                               | 필요 (회원가입 후 발급)                                      |
| **설치 복잡도** | 간단 (바로 사용)                                     | 중간 (API 키 설정 필요)                                      |
| **데이터 보안** | 높음 (로컬 저장)                                     | 중간 (외부 서버)                                             |
| **네트워크**    | 없음                                                 | 있음 (인터넷 연결 필요)                                      |
| **검색 성능**   | 기본적                                               | 고성능 (의미적 검색)                                         |
| **검색 방식**   | 키워드 기반                                          | AI 기반 의미적 검색                                          |
| **메모리 관리** | 수동적 (중복 데이터 저장)                            | 자동 분류 및 정리 (유사 정보 통합 저장 → 중복 제거)          |
| **사용 비용**   | 무료                                                 | API 사용량에 따라 과금 (10,000개 memories / 1,000건/월 무료) |
| **적합한 용도** | 개인정보 저장, 관계 데이터, 기본 메모, 오프라인 환경 | 긴 대화 내용, 복잡한 문서, 프로젝트 데이터, 의미적 검색      |
| **장점**        | 즉시 사용 가능, 데이터 보안, 네트워크 독립적 , 무료  | 고성능 검색, 자동 분류, 관련성 점수, 클라우드 백업           |
| **단점**        | 제한적 검색 기능, 수동 관리, 확장성 부족             | API 키 필요, 인터넷 의존, 비용 발생, 개인정보 외부 전송      |

> 💡 **Mem0 MCP는 홈페이지에서도 메모리를 관리할 수 있습니다.**

## 🚀 설치 및 설정

### 사전 준비

1. Node.js 18.0+ 설치
2. Claude Desktop 최신 버전 설치
3. Mem0 API 키 발급 ([https://mem0.ai](https://mem0.ai) 에서 가입 후 발급)

### 설정 방법

**1. Claude Desktop 설정 열기**

Settings > Developer > Edit Config

**2. 다음 설정을 `claude_desktop_config.json` 파일에 추가**

```json
{
  "mcpServers": {
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"],
      "description": "Knowledge Graph Memory - 안정성 우선"
    },
    "mem0": {
      "command": "npx",
      "args": ["-y", "@pinkpixel/mem0-mcp"],
      "env": {
        "MEM0_API_KEY": "your_mem0_api_key_here",
        "DEFAULT_USER_ID": "your_name"
      },
      "description": "Mem0 MCP - 고성능 검색 우선"
    }
  }
}
```

**3. API 키 설정**

`your_mem0_api_key_here` 부분을 실제 API 키로 교체하세요.

**4. Claude Desktop 재시작**

## 🧪 하이브리드 활용 전략

### 🛡️ Knowledge Graph Memory 사용 예시

- **개인정보**: "내 이름은 김연수이고, 프론트엔드 개발자야"
- **관계 데이터**: "우영우는 내 동료이고 UX 디자이너야"
- **기본 설정**: "코드 리뷰에서는 가독성을 우선시해"

### 🚀 Mem0 MCP 사용 예시

- **복잡한 문서**: "리서치 결과와 분석 내용을 메모리에 추가"
- **의미적 검색**: "AI와 관련된 모든 기억을 찾아줘"

## 💡 실제 활용 사례

두 시스템을 동시에 사용하면 다음과 같은 워크플로우가 가능합니다:

1. **개인 프로필 설정** (Knowledge Graph Memory)

   - 기본 정보, 선호 스타일, 작업 방식 저장

2. **프로젝트 컨텍스트 저장** (Mem0 MCP)

   - 프로젝트 요구사항, 회의록, 기술 스택 정보

3. **대화 연속성 유지**
   - 새 대화방에서도 이전 컨텍스트를 기반으로 대화 시작
   - 반복적인 설명 없이 작업 진행 가능
