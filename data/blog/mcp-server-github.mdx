<b>GitHub MCP Server</b>를 통해 AI 기반 코드 관리 및 저장소 작업을 자동화할 수 있습니다.

이 글에서는 Claude Desktop과 GitHub를 MCP(Model Context Protocol)로 연결하는 방법과 실제 사용 경험을 공유합니다.

---

## ⚙️ 설치 및 설정

### 📋 사전 준비사항

GitHub MCP 서버를 사용하기 위해서는 다음 준비가 필요합니다.

**1. Docker Desktop 설치 및 실행**

- [Docker Desktop 다운로드 페이지](https://www.docker.com/products/docker-desktop/)에서 운영체제에 맞는 버전 다운로드
- 설치 후 Docker Desktop 실행
- 터미널에서 `docker --version` 명령어로 설치 확인 (계정 가입 없이도 사용 가능)

**2. GitHub Personal Access Token 생성**

GitHub API 접근을 위한 토큰이 필요합니다.

1. GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
2. "Generate new token" 클릭
3. 필요한 권한 설정:
   - <b>Contents</b>: Read and Write (저장소 파일 읽기/쓰기)
   - <b>Issues</b>: Write (이슈 생성/수정)
   - <b>Pull requests</b>: Write (PR 생성/수정)
   - <b>Metadata</b>: Read (저장소 메타데이터 조회)
4. 생성된 토큰 복사 및 안전하게 보관

**3. Claude Desktop 설치**

1. [Claude 다운로드 페이지](https://claude.ai/download)에서 데스크톱 앱 다운로드
2. 설치 후 Claude Pro 계정으로 로그인

### 🔧 Claude Desktop Config 설정

운영체제별 설정 파일 위치는 다음과 같습니다.

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`

**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

**Linux:** `~/.config/Claude/claude_desktop_config.json`

설정 파일에 다음 내용을 추가합니다.

```json
{
  "mcpServers": {
    "github": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "-e",
        "GITHUB_PERSONAL_ACCESS_TOKEN",
        "ghcr.io/github/github-mcp-server"
      ],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "YOUR_TOKEN_HERE"
      }
    }
  }
}
```

### 🔄 연결 확인 및 재시작

1. 설정 파일 저장 후 <b>Claude Desktop 완전히 종료</b>
2. <b>Docker Desktop이 실행 중인지 확인</b>
3. Claude Desktop 다시 시작
4. 연결 테스트: "내 GitHub 저장소 목록을 보여줘" 메시지 전송

## 🛠️ 주요 기능

GitHub MCP 서버를 통해 다음과 같은 작업을 수행할 수 있습니다.

### ✅ 저장소 관리

- **저장소 조회**: 본인 및 조직의 저장소 목록 확인
- **파일 읽기/쓰기**: 저장소 내 파일 내용 조회 및 수정
- **브랜치 관리**: 새 브랜치 생성 및 전환
- **커밋 생성**: 파일 변경사항 커밋

### 📝 이슈 관리

- **이슈 조회**: 오픈/클로즈된 이슈 목록 확인
- **이슈 생성**: 새로운 이슈 작성 및 등록
- **이슈 수정**: 기존 이슈 내용 업데이트
- **댓글 작성**: 이슈에 댓글 추가

### 🔀 Pull Request

- **PR 조회**: Pull Request 목록 및 상세 정보 확인
- **PR 생성**: 새로운 Pull Request 작성
- **리뷰 관리**: PR 리뷰 댓글 작성 및 승인

### 기본 사용법

GitHub MCP를 연결하면 Claude에게 다음과 같이 요청할 수 있습니다.

- `"내 GitHub 저장소 목록을 보여줘"`
- `"Travel_Scheduling_Planner 저장소의 README.md 파일을 읽어줘"`
- `"오픈된 이슈들을 보여줘"`
- `"새로운 브랜치를 만들어줘: feature/github-mcp-integration"`
- `"새로운 이슈를 만들어줘: 버그 수정 필요"`

### 2. VSCode Copilot Agent로 Git 작업 자동화 테스트

figma mcp를 테스트했던 레포 [figma_mcp 프로젝트](https://github.com/yeonsu-k/figma_mcp.git)에서 VSCode Copilot Agent를 사용해 Git 작업을 자동화

**테스트 과정:**

먼저 Copilot Agent에게 프로젝트 상태를 분석하고 GitHub에 커밋해달라고 요청했습니다. Agent는 다음 작업들을 자동으로 수행했습니다:

1. **프로젝트 구조 파악** - 워크스페이스를 분석해 Nuxt.js 3 + TypeScript 기반 프로젝트임을 인식
2. **Git 상태 확인** - 17개 파일이 준비된 상태임을 확인
3. **적절한 커밋 메시지 작성** - 프로젝트 특성을 반영한 명확한 한글 커밋 메시지 생성
4. **GitHub 푸시** - 원격 저장소에 성공적으로 푸시 완료 (17개 파일, 15,729줄 추가)

사람이 직접 하면 여러 단계를 거쳐야 하는 작업을 단 한 번의 요청으로 처리했습니다. 특히 커밋 메시지를 프로젝트 컨텍스트에 맞게 자동으로 작성해주는 점이 매우 편리했습니다.

### 🤔 GitHub MCP의 아쉬운 점

Claude Desktop에서 GitHub MCP는 <b>Agent 모드</b>를 통해서만 접근할 수 있습니다. 문제는 실행 속도가 상당히 느리다는 점입니다. 간단한 작업도 응답을 받기까지 시간이 오래 걸려 답답함을 느낄 수 있습니다.

특히 <b>커밋 메시지나 PR 메시지 작성</b> 같은 작업은 GitHub MCP보다 <b>VSCode의 Copilot</b>을 사용하는 것이 훨씬 효과적입니다. Copilot은 컨텍스트를 빠르게 파악하고 적절한 메시지를 즉시 제안해줍니다.

### ✨ VSCode Copilot Agent로 이슈 생성

Copilot Agent는 이슈를 생성할 때 단순히 제목과 설명만 작성하는 것이 아니라, <b>기능을 체계적으로 구현할 수 있도록 구체적인 단계별 작업 계획</b>을 제시해줍니다.

**Copilot Agent가 생성한 이슈의 특징:**

- 구현해야 할 기능을 단계별로 세분화
- 각 단계마다 구체적인 작업 내용 명시
- 개발자가 즉시 작업에 착수할 수 있는 명확한 가이드 제공

### 🚀 Copilot Coding Agent: AI 개발자의 등장

더 나아가 <b>Copilot Coding Agent</b>는 이슈 생성을 넘어 실제 개발까지 자동화합니다.

**작동 방식:**

1. GitHub 이슈를 스스로 분석
2. 필요한 코드 변경사항을 직접 작성
3. PR(Pull Request)을 자동으로 생성 및 제출

말 그대로 <b>독립적인 AI 개발자</b> 역할을 수행하는 셈입니다.

**사용 조건**

- <b>Copilot Pro+</b> 또는 <b>Copilot Enterprise</b> 플랜 필요
- Enterprise 사용자는 조직 관리자의 기능 활성화 필요

**제약 사항**

실제로 테스트해본 결과, 몇 가지 제약이 있었습니다:

- 개인 저장소에서는 사용 불가
- Copilot Pro 플랜으로는 팀 저장소에서도 사용 불가
- <b>Enterprise 플랜</b>이 있어야 실제 활용 가능

아직은 접근성이 제한적이지만, 팀 개발 환경에서 반복적인 이슈를 처리할 때 유용하게 활용할 수 있을 것으로 보입니다.

[Access to Private Organization Repositories in GitHub MCP Server](https://github.com/github/github-mcp-server/issues/153)
