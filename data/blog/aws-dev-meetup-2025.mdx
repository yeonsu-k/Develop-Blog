AWS 개발자 모임으로 Claude Code를 어떻게 활용하면 좋을지, 토큰을 어떻게 아낄 수 있을지에 대한 팁을 얻을 수 있었다.

## 🎤 Claude Code 실습 (2025-09-18)

> 2025.09.18(목) PM 07:00 ~ 09:00

Claude Code를 실제로 써보며 유용한 설정과 팁을 공유한 세션이었다.

### SuperClaude — .claude 폴더 세팅

**.claude 폴더에 md 파일로 시스템 프롬프트를 관리**하는 방식을 소개했다. 프로젝트 컨텍스트를 Claude에게 사전에 주입하는 방법으로, 매 세션마다 반복 설명하는 수고를 줄여준다.

시스템 프롬프트를 개선할 때는 Claude에게 직접 검토를 맡기는 방식도 공유됐다:

> "시스템 프롬프트를 수정할 건데, 보완할 부분은 없는지 체크하고 개선된 문장을 영어로 입력해주세요."

`#` 을 앞에 붙이면 메모리 저장용 내용이 된다. CLAUDE.md에 저장되므로 파일을 직접 수정해도 된다.

### 추천 명령어

- **`/release-notes`**: 릴리즈 노트 확인
- **`/clear`**: 기억 초기화 = 새 세션 시작
- **`/compact`**: 현재 세션 내용 요약, 전체 맥락 파악에 유용
- **`/export`**: 파일 또는 클립보드로 내보내기
- **`/model`**: 모델 선택
- **`ultrathink`**: 프롬프트 속 숨겨진 트리거. 깊은 사고가 필요한 요청에 포함하면 효과적

### 병렬 실행

git worktree를 활용해 Claude Code 세션을 병렬로 실행하는 방법도 소개됐다. 독립적인 작업을 동시에 진행할 수 있어 효율이 높아진다.

### MCP 토큰 절약

<b>MCP는 필요한 것만 연결해서 쓰면 토큰을 아낄 수 있다.</b> 모든 MCP를 켜두면 컨텍스트를 불필요하게
소모한다고 한다.

세션 마지막에 나온 한 마디: "그냥 공식 문서 한 번씩 읽어보자…" — Claude Code를 쓰면서 공식 문서를 제대로 읽지 않고 쓰는 경우가 많은데, 기본기를 다시 보게 됐다.

## 💡 인상 깊었던 것

Claude Code를 직접 써보면서 토큰을 어떻게 아낄수 있을까, 원하는 응답을 얻기 위해 어떤 설정을 활용할 수 있을까에 대한 고민이 조금 해소됐다.
`ultrathink` 키워드나 MCP 토큰 절약 팁처럼 처음 알게된 것들도 있어서, 공식 문서를 꼼꼼히 읽어봐야겠다고 생각했다.

클로드 응답이 끝나면 알림이 오게 하기, 시스템 프롬프트를 파일로 관리하는 방법은 꽤 유용할 것 같아서 적용해봐야겠다.
