복잡한 문제를 단계별로 분석하고 해결하는 Sequential-thinking MCP 서버는 **사고 과정을 시각화**하고, **중간에 방향을 수정**하거나 **가설을 검증**할 수 있게 해줍니다.

단순한 "단계별 답변"이 아니라 **사고하는 과정을 관리하고 개선하는 메타 도구**입니다.

---

## 📋 설치 및 연결

`claude_desktop_config.json` 파일에 다음 설정을 추가하세요.

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`

**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "sequential-thinking": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sequential-thinking"]
    }
  }
}
```

설정 후 Claude Desktop App을 재시작하세요.

---

## 🎯 핵심 기능

- **🔢 구조화된 사고:** 복잡한 문제를 명확한 단계로 분해
- **🔄 동적 수정:** 중간에 이전 단계를 재검토하고 수정 가능
- **🌲 분기 탐색:** 여러 접근법을 동시에 탐색하고 비교
- **📈 적응적 계획:** 진행하면서 전체 계획을 동적으로 조정

---

## 📊 일반 AI vs Sequential-thinking AI 비교

| 구분            | 일반 AI                  | Sequential-thinking AI |
| --------------- | ------------------------ | ---------------------- |
| **사고 과정**   | 한 번에 완성된 답변 생성 | 단계별 사고 과정 노출  |
| **수정 가능성** | 처음 답변이 최종         | 중간에 방향 전환 가능  |
| **접근 방법**   | 하나의 방향으로만 진행   | 여러 방향 동시 탐색    |
| **답변 길이**   | 정해진 답변 길이         | 필요에 따라 확장/축소  |
| **투명성**      | 결과 중심 (블랙박스)     | 과정 중심 (화이트박스) |

### 3. 분기 탐색

## 💡 효과적인 활용법

### ✅ 언제 사용하면 좋을까요?

- 복잡하고 다단계적인 문제 해결이 필요할 때
- 중간에 방향 전환이 필요할 수 있는 상황
- 사고 과정 자체를 명확히 보고 싶을 때
- 여러 대안을 체계적으로 비교 검토할 때

### ⚠️ 주의사항

- 간단한 질문에는 과도할 수 있음
- 복잡한 문제일수록 더 효과적
- 구체적인 단계 수를 명시하면 더 좋은 결과

## 🎯 효과적인 질문 방법

### 상황별 최적 질문법

**기술적 의사결정:**

```
"[기술A] vs [기술B] 중 우리 프로젝트에 적합한 것을 3단계로 분석해서 선택해줘"
```

**프로젝트 계획:**

```
"[프로젝트명] 개발 계획을 세우는데, 중간에 요구사항이 바뀔 수 있으니 유연하게 5단계로 계획해줘"
```

**문제 해결:**

```
"[발생한 문제]를 해결하기 위한 방법을 여러 각도에서 분석하고 최선의 해결책을 찾아줘"
```

**성능 개선:**

```
"웹사이트 로딩 속도를 개선하는 전략을 4단계로 수립하되, 각 단계에서 여러 최적화 기법을 비교해줘"
```

### 💡 질문 품질 높이는 팁

#### 1. 맥락 제공하기

```
❌ "컴포넌트 설계해줘"
✅ "사용자 대시보드의 차트 컴포넌트를 설계하는데, 재사용성과 성능을 고려해서 3단계로 계획해줘"
```

#### 2. 구체적인 제약 조건 명시

```
❌ "최적화해줘"
✅ "모바일 성능을 고려해서 번들 크기를 줄이는 최적화 전략을 5단계로 세워줘"
```

#### 3. 목표 명확히 하기

```
❌ "도구 추천해줘"
✅ "팀 협업 효율성을 높이기 위한 개발 도구를 비교 검토해서 선택해줘"
```

### 🎪 마스터 질문 공식

```
"[구체적 상황/문제]를 [N]단계로 [분석/계획/해결]하되,
[유연성 요구사항]하고 [비교/검토 요구사항]해줘"
```

**완벽한 예시:**

```
"신규 프로젝트에서 마이크로프론트엔드 아키텍처 도입을 5단계로 계획하되,
중간에 기존 모놀리식이 더 적합하다고 판단되면 방향을 바꿀 수 있도록 하고,
여러 마이크로프론트엔드 프레임워크도 비교 검토해줘"
```
