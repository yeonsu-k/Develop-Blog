AI와 함께 작업하는 방식이 빠르게 진화하고 있습니다. 단순한 질문과 답을 넘어서, AI가 우리의 개발 워크플로우에 깊숙이 통합되는 시대가 왔습니다.

최근 <b>MCP(Model Context Protocol)</b>라는 기술을 통해 Claude를 다양한 서비스들과 연결하여 완전히 새로운 AI 워크플로우를 구축했습니다. 이 글에서는 그 과정과 경험을 공유하려고 합니다.

## 🔗 MCP(Model Context Protocol)란?

<b>MCP(Model Context Protocol)</b>는 AI 모델이 외부 시스템과 안전하고 표준화된 방식으로 상호작용할
수 있도록 하는 프로토콜입니다.

## 💡 왜 MCP를 도입했나?

개발 과정에서 Claude를 사용하면서 느꼈던 한계점들과 해결하고 싶었던 문제들이 있었습니다.

### 기존 Claude 활용의 문제점

- **컨텍스트 단절**: 각 대화 세션마다 프로젝트 배경정보를 반복 제공해야 함
- **수동 작업 의존**: AI 제안사항을 수동으로 복사하여 적용해야 함
- **도구 분산**: IDE, GitHub, Notion, Figma 등을 개별적으로 관리
- **일회성 상호작용**: 지속적인 작업 컨텍스트 유지가 어려움

### MCP를 통한 해결 방향

```
Claude Desktop ←→ MCP ←→ [GitHub, Notion, Figma, Filesystem, ...]
```

- **통합 워크플로우**: 하나의 Claude Desktop으로 모든 도구 제어
- **지속적 컨텍스트**: 프로젝트 전체 상황을 지속적으로 파악
- **자동화된 실행**: Claude가 직접 파일 수정, API 호출 가능
- **지능적 학습**: 프로젝트 히스토리와 패턴 기반 제안

## 🛠️ 연결한 서비스들

Claude에 추가적으로 필요하다고 느낀 <b>8개의 기능</b>들을 MCP를 통해 연결하여 통합된 AI 중심 워크플로우를 구축했습니다.

### 📁 개발 인프라

- **Notion**: 문서 작성, 데이터베이스 관리, 프로젝트 문서화
- **Context7**: 최신 개발 문서와 라이브러리 정보 실시간 검색

### 🧠 지식 관리

- **GitHub**: 저장소 관리, 이슈/PR 처리, 코드 검색, 브랜치 관리
- **Figma**: 디자인 파일 조회 및 컴포넌트 관리

### 🤖 AI 도구

- **Filesystem**: 로컬 파일 시스템 직접 접근 및 코드 생성/수정
- **Sequential-thinking**: 복잡한 문제를 단계별로 분석하고 해결

### 🎨 디자인 & 테스팅

- **Knowledge Graph Memory**: 프로젝트 정보와 학습 내용 지속적 저장
- **Mem0**: GPT Memory와 유사한 컨텍스트 메모리 시스템
- **Playwright**: 브라우저 자동화 및 E2E 테스트
