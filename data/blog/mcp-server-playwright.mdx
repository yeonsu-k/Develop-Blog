<b>Playwright</b>는 Microsoft에서 개발한 브라우저 자동화 및 테스트 도구로, 웹 애플리케이션의
<b>E2E(End-to-End) 테스트</b>를 자동화하는데 사용됩니다.

다양한 브라우저(Chrome, Firefox, Safari)에서 일관된 테스트가 가능하며, 사용자 인터랙션, 네트워크 요청, 페이지 스크린샷 등 다양한 작업을 자동화할 수 있습니다. Claude와 Playwright를 MCP로 연결하면 자연어 프롬프트만으로 복잡한 브라우저 테스트 시나리오를 실행할 수 있습니다.

---

## ⚡ 주요 기능

- **다중 브라우저 지원**: Chrome, Firefox, Safari 등 다양한 브라우저에서 일관된 테스트 실행
- **자동화된 E2E 테스트**: 웹 애플리케이션의 사용자 흐름과 기능을 완전히 테스트
- **스크린샷 & PDF 생성**: 페이지 캡처 및 문서 저장 기능
- **네트워크 추적**: HTTP 요청/응답 모니터링 및 검증
- **콘솔 로그 수집**: 브라우저 콘솔 로그 추적 및 분석

---

## 🛠️ 설치 및 설정

### Claude Desktop 설정

`claude_desktop_config.json` 파일에 다음 설정을 추가하세요.

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@executeautomation/playwright-mcp-server"]
    }
  }
}
```

### Cursor에 연결하기

1. Cursor를 실행한 후, Settings(설정)으로 이동합니다.
2. 상단 우측의 "Add new global MCP server"를 클릭합니다.
3. 새로 열린 mcp.json 편집 창에 다음 내용을 입력합니다.

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

## 🔄 연결 확인 및 재시작

1. Claude Desktop를 완전히 종료합니다.
2. Node.js가 설치되어 있는지 확인하고, 없는 경우 설치합니다.
3. Claude Desktop를 다시 시작합니다.

## 🎯 사용 방법

Playwright MCP를 사용하면 브라우저 조작과 E2E 테스트를 자동화할 수 있습니다.

### 예시 프롬프트

다음과 같은 자연어 프롬프트로 복잡한 테스트 시나리오를 실행할 수 있습니다.

```markdown
1. naver.com 페이지로 이동하세요.
2. 검색창에 "서울날씨"를 입력하고 Enter 키를 누르세요.
3. 현재 페이지의 스크린샷을 찍고 "서울날씨" 이름으로 저장하세요.
4. 현재 온도, 체감 온도, 습도, 풍향, 미세먼지 정보를 확인하세요.
5. naver.com 페이지로 이동하세요.
6. 검색창에 "대전날씨"를 입력하고 Enter 키를 누르세요.
7. 현재 페이지의 스크린샷을 찍고 "대전날씨" 이름으로 저장하세요.
8. 대전의 현재 온도, 체감 온도, 습도, 풍향, 미세먼지 정보를 확인하세요.
9. 서울과 대전의 날씨 정보를 비교해주세요.
```

---

## 💻 사용 경험

네이버에서 서울과 대전의 날씨 정보를 검색 후 비교하는 테스트를 실행해보았습니다.

<b>Chromium 기반</b>으로, 자동으로 네이버 웹사이트에 접속하여 검색창에 "서울날씨"와 "대전날씨"를
각각 입력하고, 결과를 확인할 수 있었습니다. 각 도시의 현재 온도, 체감 온도, 습도, 풍향, 미세먼지
정보를 성공적으로 추출하여 두 도시의 날씨를 비교할 수 있었습니다.

테스트 시나리오를 작성하는 것만으로 Claude가 자동으로 브라우저를 조작하고 결과까지 정리해주어 매우 편리했습니다. 수집한 데이터를 <b>노션이나 리포트</b> 형태로 정리하는 용도로도 활용할 수 있을 것 같습니다.

## 🔧 고급 기능

Playwright MCP는 다양한 고급 기능을 제공합니다.

- **브라우저 컨텍스트 추적**: 브라우저 콘솔 로그, 네트워크 요청 및 응답 추적 기능
- **기록 및 재생**: 테스트 실행 과정을 기록하고 다시 재생하는 기능
- **상세한 오류 보고**: 테스트 실패 시 상세한 오류 정보와 스크린샷 제공
- **병렬 테스트**: 여러 브라우저에서 동시에 테스트 실행 가능
- **자동 코드 생성**: 테스트 시나리오를 기반으로 테스트 코드 자동 생성
