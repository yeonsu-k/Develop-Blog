---

## 공통 스타일 규칙

**톤앤매너 (모든 글 공통)**
- 인트로는 존댓말 없는 서술체 2~3문장. 문제/목적/맥락을 제시
- 본문 전체 존댓말 없는 간결한 서술체
- `<b>` 태그로 핵심 키워드·강조 표현
- 이모지 헤딩: `## ⚙️`, `## 🛠️`, `## 💻`, `## 📚` 등 섹션 성격에 맞게
- `---` 구분선으로 인트로 / 본문 / 참고자료 구분
- 코드 블록에 언어 태그 (json, bash, css 등)
- `>` 인용구는 팁·주의사항에 사용

**이미지 삽입 형식:**
```mdx
<div className="flex flex-col items-center justify-center gap-3 md:flex-row md:gap-5">
  <img
    src="/static/images/blog/{폴더}/{파일}.png"
    alt="{설명}"
    className="!mt-0 !mb-0 w-full rounded-lg shadow-sm sm:w-11/12 md:w-[50%]"
  />
</div>
```
이미지 1장이면 `md:w-[50%]` 제거, 이미지 없으면 블록 생략

**Frontmatter (모든 글 공통)**
```yaml
---
title: '{제목}'
date: '{날짜}'
tags: ['{태그1}', '{태그2}']
draft: false
summary: '{1줄 요약}'
images: []
authors: ['default']
---
```

## 데이터 소스 안내

- **노션 기반 글**: 노션 MCP(`notion_retrieve_block_children`)로 해당 페이지 ID의 블록을 읽어 내용 파악

- **두 소스 합치는 경우**: 노션 내용을 뼈대로, 로컬 파일로 디테일 보완

작성할 글의 유형(A/B)과 데이터 소스(노션 페이지 ID 또는 파일 경로)를 알려주세요.
